# Obsidian 图片粘贴管理神器：Image Markdown Paste 插件完全指南

> 解决 Obsidian 图片管理混乱、跨平台兼容性差的痛点，让你的笔记图片管理井井有条

---

## 一、为什么要开发这个插件？

### 1.1 Obsidian 原生图片粘贴的问题

Obsidian 作为目前最受欢迎的本地知识库工具，其默认的图片粘贴行为却存在几个让人头疼的问题：

**问题 1：图片存放混乱**
- 所有粘贴的图片都堆放在 `attachments` 目录下
- 文件命名格式为 `Pasted image 20260312101246.png`，难以识别归属
- 长期使用后，attachments 文件夹变成"图片垃圾堆"

**问题 2：图片引用格式不标准**
- Obsidian 默认使用 WikiLink 格式：`[[image.png]]` 或仅文件名
- 这种格式在其他 Markdown 编辑器（Typora、VS Code、GitHub）中无法正常显示
- 分享笔记时图片全部"失联"

**问题 3：文件重命名/移动后链接失效**
- 重命名文档后，相关图片不会同步更新
- 移动文档到其他文件夹，图片引用路径失效
- 手动修复链接费时费力

### 1.2 实际使用场景的困扰

想象一下这些场景：

> 📌 **场景 A**：你在 Obsidian 中整理了一篇技术笔记，粘贴了 20 张截图。一个月后你想把这篇笔记分享到博客，发现所有图片都显示不出来，因为路径格式不兼容。

> 📌 **场景 B**：你决定重命名一篇笔记让其标题更准确，结果发现笔记里的 10 张图片引用全部失效，不得不一张张重新关联。

> 📌 **场景 C**：你的 `attachments` 文件夹已经积累了 500+ 张图片，想清理无用的图片却无从下手，因为根本不知道哪些图片属于哪篇笔记。

**Image Markdown Paste 插件就是为了解决这些问题而生的。**

## 二、插件核心功能

### 2.1 功能概览

| 功能 | 说明 |
|------|------|
| **自定义保存路径** | 支持变量动态生成图片保存路径，如 `assets/{filename}/{date}` |
| **标准 Markdown 格式** | 使用 `![alt](相对路径)` 格式，跨平台兼容 |
| **重命名同步** | 文档重命名时自动同步更新图片名称 |
| **移动同步** | 文档移动时自动将图片移动到新的配置路径 |

### 2.2 支持的变量

在配置保存路径时，可以使用以下变量实现动态路径：

| 变量 | 说明 | 示例输出 |
|------|------|----------|
| `{filename}` | 当前文档文件名（不含扩展名） | `我的笔记` |
| `{filepath}` | 当前文档完整路径（不含扩展名） | `projects/我的笔记` |
| `{folder}` | 当前文档所在文件夹名称 | `projects` |
| `{folderpath}` | 当前文档所在文件夹完整路径 | `projects` |
| `{date}` | 当前日期 | `20260312` |
| `{time}` | 当前时间 | `101246` |
| `{datetime}` | 当前日期时间 | `20260312-101246` |
| `{year}` | 当前年份 | `2026` |
| `{month}` | 当前月份 | `03` |
| `{day}` | 当前日期 | `12` |

## 四、插件配置与使用方法

### 4.1 基础配置

安装完成后，进入插件设置：

**设置路径**：设置 → 第三方插件 → Image Markdown Paste → 选项

**【截图位置 5】**
> 请截取插件设置页面的完整界面，显示所有配置选项

#### 推荐配置方案

**方案 A：按文档名称组织（推荐）**
```
图片保存路径: assets/{filename}
使用标准 Markdown 格式: ✅ 开启
文件名冲突时自动重命名: ✅ 开启
启用重命名同步: ✅ 开启
启用移动同步: ✅ 开启
```
**效果**：图片保存在 `assets/文档名/` 目录下

**方案 B：按日期组织**
```
图片保存路径: attachments/{year}/{month}/{date}
```
**效果**：图片按日期分层存放，如 `attachments/2026/03/20260312/`

**方案 C：与文档同目录**
```
图片保存路径: {folderpath}/{filename}_assets
```
**效果**：图片文件夹与文档紧邻存放

### 4.2 日常使用

#### 粘贴图片

配置完成后，使用方式与平时完全一样：

1. 复制任意图片（截图、浏览器右键复制图片等）
2. 在 Obsidian 编辑器中按 `Ctrl/Cmd + V` 粘贴
3. 插件会自动：
   - 将图片保存到配置的路径
   - 生成标准 Markdown 格式的引用
   - 插入到光标位置

**【截图位置 6】**
> 请截取粘贴图片后的编辑器界面，显示标准 Markdown 格式的图片引用代码

#### 转换现有文档的图片引用

如果你已有大量使用 WikiLink 格式的笔记，可以使用转换功能：

1. 打开需要转换的文档
2. 打开命令面板（Ctrl/Cmd + P）
3. 搜索并执行 "Image Markdown Paste: 转换图片引用为标准 Markdown"

**【截图位置 7】**
> 请截取命令面板中显示转换命令的界面

#### 整理已有图片

如果你想把已有文档的图片重新整理到新的路径结构：

1. 打开目标文档
2. 打开命令面板
3. 搜索并执行 "Image Markdown Paste: 整理当前文档的图片到配置路径"
4. 插件会自动将所有图片移动到配置的路径，并更新引用

## 六、高级技巧

### 6.1 配合 Git 版本控制

由于使用相对路径，笔记仓库可以完美配合 Git 管理：

```bash
# 提交到 GitHub 后，图片链接在网页端也能正常显示
git add .
git commit -m "update notes"
git push origin main
```

### 6.2 批量迁移旧笔记

如果你有大量旧笔记需要整理：

1. 配置好目标路径格式
2. 逐个打开旧笔记
3. 执行 "整理当前文档的图片到配置路径" 命令
4. 所有图片会自动归类到新的路径结构

### 6.3 多设备同步

配合 Obsidian Sync、iCloud Drive 或 Dropbox 使用时：

- 相对路径确保所有设备上图片都能正常加载
- 按文档组织的图片结构让同步更高效

## 七、常见问题

**Q: 插件会影响 Obsidian 原有的粘贴功能吗？**
A: 不会。插件只处理图片粘贴，文本粘贴保持原有行为。

**Q: 已有的 WikiLink 格式图片会失效吗？**
A: 不会。插件不会自动修改已有内容，你可以手动使用转换命令批量转换。

**Q: 卸载插件后图片还能正常显示吗？**
A: 可以。因为插件使用的是标准 Markdown 格式，卸载后图片引用依然有效。

**Q: 支持移动端吗？**
A: 支持。插件标记为全平台兼容，可在 iOS 和 Android 上使用。
